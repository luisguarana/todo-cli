  <action>
  Create todo.js at the workspace root. Implementation requirements (per decisions D-01 through D-11):

  Line 1: shebang `#!/usr/bin/env node` (D-04).
  Module system: CommonJS — use `require('fs')`, `require('path')`, `require('process')` only. No `import`, no external packages (D-02, CON-01).

  Storage path: `path.join(process.cwd(), 'todos.json')` — always relative to CWD, not __dirname (STR-01).

  Read helper: read todos.json with `fs.readFileSync`; if file does not exist (catch ENOENT), return empty array `[]` (STR-02).
  Write helper: `fs.writeFileSync` with `JSON.stringify(arr, null, 2)` for readable output.

  Command dispatch via `process.argv[2]` (the subcommand). Commands:

  `add` (CMD-01, D-07, D-10):
  - Read current items.
  - New ID = items.length === 0 ? 1 : Math.max(...items.map(i => i.id)) + 1.
  - Push `{ id: newId, text: process.argv[3] }` onto array.
  - Write file.
  - Print: `Added: <text> (ID: <N>)`.

  `list` (CMD-02, D-08, D-09):
  - Read current items.
  - If empty, print `No todos yet.` and return.
  - Otherwise print each item as `<id>. <text>` (e.g., `1. buy milk`).

  `done` (CMD-03, D-06, D-11):
  - Read current items.
  - Parse ID from process.argv[3] as integer.
  - Find item by id.
  - Remove it from array.
  - Renumber remaining items: items[i].id = i + 1 for i in 0..N-1 (D-05, D-06).
  - Write file.
  - Print: `Done: removed "<text>"`.
  - Note: no error handling in Phase 1 — that is Phase 2 scope.

  No default/help case needed in Phase 1 — that is CMD-04 / Phase 2 scope.
  </action>
