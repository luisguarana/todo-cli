# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-06-02
**Phases:** 1–2
**Total Plans:** 3

## Overview

Two phases delivered a complete zero-dependency Node.js CLI for personal to-do management. Phase 1 shipped a working tool that adds, lists, and marks tasks done with local JSON persistence. Phase 2 hardened the experience with graceful error handling and usage help so every input path produces clear feedback instead of crashes.

## Phases

### Phase 1: Core CLI

**Goal:** Users can add, list, and remove to-do items entirely from the terminal with zero setup
**Depends on:** Nothing (first phase)
**Plans:** 2/2 complete

Plans:

- [x] 01-01-PLAN.md — Implement todo.js with add, list, and done commands (Wave 1)
- [x] 01-02-PLAN.md — Add bin field to package.json for global CLI registration (Wave 1)

**Details:**
- `todo.js` implemented as a single CommonJS file using `fs`, `path`, `process` only
- `add` appends `{id, text}` to `todos.json`, auto-creates file on first run via ENOENT catch
- `list` prints each item as `<id>. <text>` or `No todos yet.`
- `done` removes item by ID and renumbers remaining items 1..N
- `package.json` bin field (`"todo": "./todo.js"`) wired for `npm link` global install
- All 7 Phase 1 requirements satisfied: CMD-01, CMD-02, CMD-03, STR-01, STR-02, STR-03, CON-01

## Milestone Summary

**Key Decisions:**

- todos.json stored in CWD (not home dir) for per-directory project isolation
- `done` removes items permanently — no completed-item archive, keeps list clean
- Zero external npm dependencies — portability and simplicity, installs anywhere Node runs
- Usage block printed to stdout (not stderr) — exits 0 so shell pipes work
- Not-found message includes requested ID for immediate user clarity
- Missing todos.json + `done <id>` reuses ENOENT [] return — no separate branch needed

**Issues Resolved:**

- Code review found 5 input-validation issues (CR-01–CR-05, WR-01–WR-03); all fixed before ship
- `bin` field already committed in a prior session; duplicate prevented via git diff check

**Issues Deferred:**

- None — clean ship

**Technical Debt Incurred:**
